# Infrastructure Automation with Terraform + Ansible + Bash + Powershell
This project demonstrates how to automate the provisioning and configuration of a multi-OS environment using Terraform, Ansible, Bash, and PowerShell. It creates:
- 2 Linux EC2 instances (Amazon Linux 2023)
- 1 Windows EC2 instance (Windows Server 2022)

Then uses Ansible to configure:
- Linux web servers with Nginx
- Windows web server with IIS

## Prerequisites

1. Install tools:
   - Terraform >= 1.5
   - AWS CLI configured (`aws configure`)
   - Ansible
   - Python package for WinRM:
     ```bash
     pip install pywinrm
     ```
2. Create an EC2 Key Pair in AWS (or reuse existing).
3. Know your public IP CIDR (example `203.0.113.10/32`) for secure inbound access.

## Step-by-Step

1. Move to the Terraform folder:
   ```bash
   cd cloud-security-foundation/aws/labs/three-vm-ansible/terraform
   ```

2. Create your `terraform.tfvars`:
   ```bash
   cp terraform.tfvars.example terraform.tfvars
   ```
   Edit values:
   - `ssh_key_name`: your EC2 key pair name
   - `ssh_private_key_path`: local private key path
   - `windows_admin_password`: strong temporary lab password
   - `trusted_ingress_cidr`: your public IP CIDR

3. Initialize and apply Terraform:
   ```bash
   terraform init
   terraform apply
   ```
   Terraform also generates:
   - `cloud-security-foundation/aws/labs/three-vm-ansible/ansible/inventory.ini`

4. Move to Ansible folder:
   ```bash
   cd ../ansible
   ```

5. Install required Ansible collection:
   ```bash
   ansible-galaxy collection install -r requirements.yml
   ```

6. Test connectivity:
   ```bash
   ansible linux -m ping
   ansible windows -m ansible.windows.win_ping
   ```

7. Configure Linux web servers:
   ```bash
   ansible-playbook playbooks/linux-web.yml
   ```

8. Configure Windows web server:
   ```bash
   ansible-playbook playbooks/windows-web.yml
   ```

9. Validate in browser:
   - `http://<linux-web-1-public-ip>`
   - `http://<linux-web-2-public-ip>`
   - `http://<windows-web-1-public-ip>`

## Optional One-Command Scripts

From `three-vm-ansible/`:

```bash
./scripts/deploy.sh
```

Destroy all resources:

```bash
./scripts/destroy.sh
```

## Notes

- This is a learning lab. Windows WinRM is enabled with Basic + HTTP for Ansible simplicity.
- For production, use WinRM over HTTPS, stronger hardening, SSM/private networking, and secrets management.
